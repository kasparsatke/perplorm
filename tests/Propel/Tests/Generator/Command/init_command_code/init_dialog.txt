  Set up project structure.  


A typical perpl project looks like this:

%projectDir%/                              project root directory
    │
    ├─╼ %perplDir%/                          perpl config dir (should be 'config', 'conf', 'perpl', or 'db' if to be resolved automatically by perpl commands)
    │      │
    │      ├─╼ %schemaDir%/                    Schema XML files (input for migration:diff, database:reverse, etc)
    │      │      └─╼ schema.xml                 Main schema file
    │      │
    │      ├─╼ generated-conf/                 PHP scripts to import database configuration (from config:convert and model:build)
    │      ├─╼ generated-migrations/           Migration files (target for migration:create, migration:migrate, etc)
    │      ├─╼ generated-sql/                  SQL database initialization files for sql:insert (user-generated and generated from schema.xml by sql:build)
    │      ├─╼ perpl.json                      Instance-specific configuration values (i.e. database connection)
    │      └─╼ perpl.dist.json                 Shared configuration values (i.e. main directory structure)
    │
    └─╼ %phpDir%/                            Base target directory for model:build, root of PHP namespace, typically '/src/'

You can adjust this structure later by editing the config file.
